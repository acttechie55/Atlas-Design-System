Buttons trigger interactions throughout the experience.

Primary Buttons

Use for a user's prinicipal action to completing their goal.

When To Use:

Use this button if the user's action is positive (i.e. 'Send' or 'Submit').

Code:

<button class="btn atlas-primary"></button>

When To Use:

Use this button if the user's action is negative or "dangerous" (i.e. 'Replace' or 'Delete').

Code:

<button class="btn atlas-primary-danger"></button>

Secondary Buttons

Use for non-critical actions like "Cancel", "Go Back", "Reset", "Save for Later". These actions are typically counter to people's primary goal.

Code:

<button class="btn atlas-secondary"></button>

Code:

<button class="btn atlas-secondary-danger"></button>

Tertiary Buttons

Use for actions that are not directly tied to the person's current goal. This will be used based on the particular context of its placement. It intentionally has less visual prominence than the other two because we do not want to distract from the user's goal.

Code:

<button class="btn atlas-tertiary"></button>

Disabled Buttons

Code:

Add disabled attribute to button.
